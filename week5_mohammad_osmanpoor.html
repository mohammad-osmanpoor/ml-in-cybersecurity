# Define the range of k values to test
k_values = range(5, 55, 5)  # From 5 to 50 with steps of 5

# Initialize list to store AUC scores for different k values
auc_scores = []

# Loop through each k value and calculate AUC
for k in k_values:
    # Create the LOF model with current k value
    # novelty=True allows me to use decision_function later
    detector_lof = LocalOutlierFactor(n_neighbors=k, novelty=True)

    # Train the model
    detector_lof.fit(X)

    # Calculate anomaly scores using decision_function
    # Higher scores indicate more normal instances (not anomalies)
    y_score = detector_lof.decision_function(X)

    # Calculate AUC score
    # Since higher y_score values indicate normal instances and we want to detect anomalies,
    # I use the raw scores directly with roc_auc_score
    auc = roc_auc_score(y, y_score)

    # Append the AUC score to our list
    auc_scores.append(auc)

    # Print current k and its corresponding AUC
    print(f'k = {k}, AUC score: {auc:.6f}')

✍Observations¶

Based on the Local Outlier Factor (LOF) parameter tuning results, I can see that k=20 provides the optimal performance with an AUC score of 0.672489 for this dataset. The AUC scores demonstrate a clear trend, increasing from 0.516108 at k=5 to the peak at k=20, followed by a gradual decline as k increases further. This pattern underscores the importance of selecting an appropriate neighborhood size that balances local density estimation with the global context of the data. The significant performance difference between the lowest and highest AUC scores highlights LOF's sensitivity to the choice of k, emphasizing the critical role of parameter tuning in anomaly detection tasks. Given the dataset's anomaly rate of 3.88%, the optimal k value of 20 suggests that considering approximately 0.14% of the total data points as neighbors yields the best anomaly detection capability for datasets with similar characteristics.¶

The ROC curve for k=20 indicates reasonable detection performance, achieving high true positive rates at moderate false positive rates, which is crucial in anomaly detection scenarios where the cost of false negatives can be substantial. These findings collectively demonstrate the nuanced relationship between the number of neighbors and LOF's effectiveness, underscoring the necessity of thorough parameter optimization.¶

✍Observations¶

In the analysis of Isolation Forest performance across different sample sizes, I see clear pattern of improvement as the sample size increases, with optimal performance achieved at a sample size of 2048, yielding an AUC score of 0.951657. The performance curve demonstrates a steep initial improvement from the smallest sample size of 32 (AUC: 0.823239) through 256 (AUC: 0.932429), followed by more gradual gains until reaching the peak. I think this pattern suggests that while small sample sizes fail to capture the complexity of anomalies in the dataset, there exists a point of diminishing returns beyond the optimal sample size, as evidenced by the slight performance decline at 4096 and 8192. The warning message observed when testing the 16384 sample size indicates that this value exceeds the total number of samples in the dataset (14081), causing the algorithm to automatically adjust to use all available samples. The resulting ROC curve for the optimal sample size of 2048 exhibits excellent performance with an AUC of 0.95, demonstrating the model's strong ability to distinguish between normal and anomalous instances across different threshold settings. These findings highlight the importance of properly tuning the sample size parameter in Isolation Forest to balance between capturing sufficient data complexity and avoiding overfitting, particularly for datasets with a relatively low anomaly rate of 3.88% as observed in this case.¶
