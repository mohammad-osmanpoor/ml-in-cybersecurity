# Import necessary libraries
import numpy as np
import pandas as pd
from matplotlib import pyplot as plt
import diffprivlib.models as dp
from sklearn.preprocessing import MinMaxScaler
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

# Load the dataset
data = pd.read_csv("data/income.csv")
print("Dataset loaded successfully. Shape:", data.shape)
# Verifying the shape to ensure the data is ready for processing

# Preprocess the data
# Extract features (X) and target (y) for classification
X = data.drop(['Income'], axis=1).values
y = data['Income'].values
# Scale features to [0,1] for compatibility with diffprivlib
scaler = MinMaxScaler()
X_scaled = scaler.fit_transform(X)
print("Data scaled to [0,1] range for differential privacy.")
# Scaling ensures all features are in [0,1], matching the bounds required by diffprivlib

# Split the data into training and testing sets
X_train, X_test, y_train, y_test = train_test_split(X_scaled, y, test_size=0.20, random_state=42)
print("Data split into training and testing sets. Training shape:", X_train.shape, "Testing shape:", X_test.shape)
# Using a fixed random_state ensures reproducibility of the split

# Set parameters for differentially private Naive Bayes
epsilons = np.logspace(-2, 1, 10)  # Epsilon values from 0.01 to 10
bounds = ([0] * X_scaled.shape[1], [1] * X_scaled.shape[1])  # Bounds for each feature (scaled to [0,1])
n_iterations = 5  # Number of runs per epsilon to average out randomness
accuracy_scores = []  # Store average accuracy for each epsilon
# Using 5 iterations balances accuracy and runtime, as differential privacy introduces randomness

# Run differentially private Naive Bayes for each epsilon
for eps in epsilons:
    scores = []
    for _ in range(n_iterations):
        # Initialize and fit the differentially private Gaussian Naive Bayes model
        dp_clf = dp.GaussianNB(epsilon=eps, bounds=bounds)
        dp_clf.fit(X_train, y_train)
        # Predict on the test set and calculate accuracy
        y_pred = dp_clf.predict(X_test)
        score = accuracy_score(y_test, y_pred)
        scores.append(score)
    # Average the accuracy scores for this epsilon
    avg_score = np.mean(scores)
    accuracy_scores.append(avg_score)
    print(f"Epsilon: {eps:.2f}, Average Accuracy: {avg_score:.4f}")


# Visualize epsilon vs. accuracy
plt.plot(epsilons, accuracy_scores, marker='o')
plt.xscale('log')  # Log scale for better visualization of epsilon range
plt.xlabel('Epsilon (log scale)')
plt.ylabel('Average Accuracy')
plt.title('Epsilon vs. Classification Performance (Accuracy)')
plt.show()
# This plot illustrates the privacy-utility trade-off for Naive Bayes classification

Questions that have mentioned in demo section¶

Question 1 Does Preprocessing Protect Privacy ? The preprocessing step in the demo, which scales the features of the income.csv dataset to the range [0, 1] using MinMaxScaler, does not inherently protect privacy. Scaling transforms the data to ensure all features contribute equally to the model and meet the bounds required by diffprivlib, but this transformation is deterministic and reversible if the scaling parameters (minimum and maximum values) are known. For example, an attacker with access to these parameters could reverse the scaling to recover the original values, potentially exposing sensitive information like an individuals age or hours worked per week. True privacy protection in this context comes from the differential privacy mechanism applied during model training, where noise is added to the models statistics (e.g., means and variances in Naive Bayes) based on the privacy parameter . The demo highlights this by using diffprivlibs GaussianNB with specified bounds and values, ensuring privacy through noise addition rather than relying on preprocessing alone.

Question 2: What Kind of Basic Statistical Computing Operations Are Used in Gaussian NBC? The Gaussian Naive Bayes Classifier (NBC) assumes that the features follow a Gaussian (normal) distribution for each class. To make predictions, it relies on several statistical operations. First, during training, the algorithm computes the mean and variance of each feature for each class in the dataset. For example, in the income.csv dataset, it calculates the mean and variance of features like Age and Hours per Week separately for the <=50K and >50K income classes. These statistics define the Gaussian distribution for each feature-class pair. Additionally, it calculates the prior probability of each class, which is the proportion of training samples belonging to that class (for example the fraction of samples with Income <=50K). During prediction, Gaussian NBC uses these statistics to compute the likelihood of a new sample belonging to each class. For each feature, it evaluates the Gaussian probability density function using the features value, the mean, and the variance for that class. The final prediction is made by multiplying the likelihoods (due to the naive independence assumption) and the class prior, then selecting the class with the highest posterior probability.

Question 3: What Are the Sensitivities of These Operations? In the context of differential privacy, sensitivity measures how much a statistical operations output changes when one individuals data is added or removed from the datasetthis determines how much noise must be added to protect privacy. For Gaussian NBC, the key operations are computing the mean, variance, and class priors. The sensitivity of the mean depends on the features bounds; since the features in the demo are scaled to [0, 1], adding or removing one sample can change the mean by at most 1/n (where n is the number of samples in the class), so the sensitivity is 1/n. The variance calculation involves squared differences from the mean, making its sensitivity more complex, but for bounded data in [0, 1], its typically bounded by 1/n as well, though the exact value depends on the implementation. The class priors sensitivity is also 1/n, as adding or removing one sample changes the class proportion by at most 1/n. In diffprivlib, these sensitivities are used to add noise (for example, Laplacian or Gaussian noise) to the mean and variance, scaled by the privacy parameter . Smaller values require more noise, which can reduce accuracy.

# Import necessary libraries
import numpy as np
import pandas as pd
from matplotlib import pyplot as plt
import diffprivlib.models as dp
from sklearn.preprocessing import MinMaxScaler
from sklearn.metrics import silhouette_score
from sklearn.decomposition import PCA

# Load the dataset
data = pd.read_csv("data/income.csv")
print("Dataset loaded successfully. Shape:", data.shape)
# Verifying the shape to ensure the data is ready for processing

# Preprocess the data
# Remove the target column since K-means is unsupervised
X = data.drop(['Income'], axis=1).values
# Scale features to [0,1] for compatibility with diffprivlib
scaler = MinMaxScaler()
X_scaled = scaler.fit_transform(X)
print("Data scaled to [0,1] range for differential privacy.")
# Scaling ensures all features are in [0,1], matching our bounds

# Set parameters
K = 5  # Number of clusters (I can experiment with this value to find a reasonable choice)
epsilons = np.logspace(-2, 1, 10)  # Epsilon values from 0.01 to 10
# Corrected bounds: (lower, upper) as a tuple of two lists for all features
bounds = ([0] * X_scaled.shape[1], [1] * X_scaled.shape[1])
n_iterations = 5 # 10  # Multiple runs to account for randomness in differential privacy
silhouette_scores = []  # Store average scores for each epsilon
# Fixed bounds to (lower, upper) format required by diffprivlib

# Run differentially private K-means for each epsilon
for eps in epsilons:
    scores = []
    for _ in range(n_iterations):
        # Initialize and fit the differentially private K-means model
        dp_kmeans = dp.KMeans(n_clusters=K, epsilon=eps, bounds=bounds)
        dp_kmeans.fit(X_scaled)
        # Predict cluster labels
        labels = dp_kmeans.predict(X_scaled)
        # Calculate silhouette score to evaluate clustering quality
        score = silhouette_score(X_scaled, labels)
        scores.append(score)
    # Average the scores for this epsilon
    avg_score = np.mean(scores)
    silhouette_scores.append(avg_score)
    print(f"Epsilon: {eps:.2f}, Average Silhouette Score: {avg_score:.4f}")

✍ Analysis of Results¶

I have implemented a differentially private K-means clustering algorithm using the IBM Differential Privacy Library (diffprivlib) on the income.csv dataset, which contains 32,561 records and 5 features after removing the target Income column. The primary objective was to investigate how the privacy parameter (epsilon) impacts clustering performance, measured through the silhouette score, and to visualize the results to understand the privacy-utility trade-off. I preprocessed the dataset by scaling all features to the range [0, 1] using MinMaxScaler to meet diffprivlibs bounds requirement. I fixed the number of clusters at K = 5, as suggested by the assignment, and tested 10 epsilon values ranging from 0.01 to 10.00 on a logarithmic scale. To account for the randomness introduced by differential privacy, I ran 5 iterations for each value, averaging the silhouette scores to ensure reliable results while balancing computational efficiency.

The results revealed a range of silhouette scores across the tested epsilon values, with scores fluctuating between 0.2335 (at = 0.05) and 0.3385 (at = 0.02). These values indicate moderate clustering quality, as silhouette scores closer to 1 would suggest well-separated clusters, while my results suggest some overlap between clusters. Interestingly, there was no clear monotonic trend in the scores as increased. For instance, the highest silhouette score of 0.3385 occurred at = 0.02, a very small value indicating strong privacy, while the lowest score of 0.2335 was at = 0.05, not the smallest . At the largest of 10.00, which offers the least privacy, the score was 0.3149, not the highest. This lack of a consistent trend suggests that the noise added by differential privacy affects clustering in a complex way. Small values introduce more noise, which can disrupt cluster formation, but the randomness of the noise sometimes aligns with the datas natural structure, as seen at = 0.02. Larger values, which are closer to non-private K-means, did not always yield better performance, possibly due to the datasets inherent complexity or the choice of K.

The privacy-utility trade-off, a key concept in differential privacy, was not as pronounced as expected in this experiment. Differential privacy protects data by adding noise, with smaller values providing stronger privacy at the potential cost of utility (clustering quality), and larger values offering less privacy but potentially better utility. However, my results showed relatively stable silhouette scores between 0.23 and 0.34 across all values, with the best performance at a very small . This indicates that the impact of on clustering quality is less predictable than anticipated, possibly because the noises effect varies with each run, or because the income.csv dataset does not have strongly distinct clusters to begin with. The scatter plot revealed five clusters with noticeable overlap, particularly in the central region, which aligns with the moderate silhouette score of 0.3038 at = 1.0. For even faster execution, I could have sampled the dataset or used a simpler metric like Within-Cluster Sum of Squares (WCSS), but I prioritized the silhouette score for its interpretability. This experiment taught me the challenges of applying differential privacy to unsupervised learning.
